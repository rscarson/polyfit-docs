Glossary

A glossary of mathematical concepts behind polynomial fitting, with visual illustrations and links to further reading.

A basis is the set of functions used to construct the polynomial model.
Each has their own strengths and weaknesses, and the choice of basis can significantly impact the performance and accuracy of the fit.
Examples of properties include orthogonality, numerical stability, and resistance to outliers.

Monomial Basis
y(x) = 1.82x³ + 286.38x² - 1.15e5x + 1.47e6

When you say polynomial, most people think of the monomial basis. It's recognizable, universal, and great for displaying or communicating a fit to people.

However, it can be numerically unstable for high-degree polynomials or large input ranges, leading to poor results.

Prefer Chebyshev, which can be converted to monomial form for display.

  • Input Domain None — the function will use the given x-values without modification.
  • Resistance to Outliers Poor. Extreme values or spikes in the data set can significantly affect the fit.
  • Numerical Stability Low. Susceptible to numerical instability, especially for high-degree polynomials or large input ranges.
  • Calculus Support Full support for symbolic differentiation, integration, and exact root finding. See [features#calculus].
  • Orthogonal No — functions like [CurveFit::spectral_energy_filter] are unavailable.
Chebyshev Basis
T[ 0..1000 -> -1..1 ], y(x) = 5.69e7·T₃(xₛ) + 3.77e8·T₂(xₛ) + 9.39e8·T₁(xₛ) + 6.20e8

Chebyshev polynomials are a set of orthogonal polynomials that provide excellent numerical stability and resistance to outliers. Their properties make them very resistant to outliers at the edges of the input range.

They are particularly well-suited for high-degree polynomial fits and large input ranges, and can be converted to monomial form for display or communication purposes, making them very versatile.

  • Input Domain [-1, 1] — input x-values are scaled and shifted to fit within this range. That transformation will be reflected in the equation.
  • Resistance to Outliers Good. Chebyshev polynomials do something called minimax approximation to minimize the maximum error, which helps reduce the impact of outliers.
  • Numerical Stability High. Chebyshev polynomials are orthogonal and well-conditioned, making them more stable for high-degree fits and large input ranges.
  • Calculus Support Supports derivative into the 2nd form, and again into the 3rd form. Both can be integrated back up to the original. Can be converted to monomial form. Exact root finding supported. See [features#calculus].
  • Orthogonal Yes. See [orthogonality].
Legendre Basis
T[ 0..1000 -> -1..1 ], y(x) = 9.11e7·P₃(xₛ) + 5.03e8·P₂(xₛ) + 9.05e8·P₁(xₛ) + 4.95e8

Legendre polynomials are a set of orthogonal polynomials that provide excellent numerical stability and resistance to outliers, similar to Chebyshev polynomials.

Compared to chebyshev polynomials, Legendre spreads the approximation error more evenly across the input range, rather than the edges.

They are particularly well-suited for high-degree polynomial fits, and can be converted to monomial form for display or communication purposes, making them very versatile.

  • Input Domain [-1, 1] — input x-values are scaled and shifted to fit within this range. That transformation will be reflected in the equation.
  • Resistance to Outliers Fair. Legendre polynomials aim to minimize the overall error across the input range, but may be less effective at handling outliers compared to Chebyshev polynomials.
  • Numerical Stability High. Legendre polynomials are orthogonal and well-conditioned, making them more stable for high-degree fits.
  • Calculus Support Supports derivativation, but not integration. Can be converted to monomial form. Root finding uses numerical approximation. See [features#calculus].
  • Orthogonal Yes. See [orthogonality].
Laguerre Basis
y(x) = -10.93·L₃(x) + 605.54·L₂(x) + 1.14e5·L₁(x) + 1.36e6

Laguerre polynomials are a set of orthogonal polynomials that are particularly well-suited for modeling data that decays exponentially, such as radioactive decay or certain financial models.

They have an infinite, but positive input domain [0, ∞), so the source range does not need to be compressed. They have good numerical stability and resistance to outliers, and can be converted to monomial form for display or communication purposes, making them versatile.

  • Input Domain [0, ∞) — input x-values shifted to fit within this range.
  • Resistance to Outliers Good. Laguerre polynomials are less sensitive to outliers due to their orthogonal nature.
  • Numerical Stability High. Legendre polynomials are orthogonal and well-conditioned, making them more stable for high-degree fits, and their 0 to ∞ domain makes them suited for large input ranges.
  • Calculus Support Supports derivativation, but not integration. Can be converted to monomial form. Root finding uses numerical approximation. See [features#calculus].
  • Orthogonal Yes. See [orthogonality].
Hermite Basis
y(x) = 0.23·He₃(x) + 71.59·He₂(x) - 5.76e4·He₁(x) + 1.47e6

Hermite polynomials are a set of orthogonal polynomials that come in two flavors:

  • Physicist's Hermite polynomials (Heₙ), used in quantum mechanics and physics.
  • Probabilist's Hermite polynomials (Hₙ), used in probability theory and statistics.
The choice between the two depends on the specific application, but both share similar properties. The primary difference is a scaling factor, allowing each variant to fit important equations in their respective fields.

They have no specific input domain restrictions, and have good numerical stability and resistance to outliers. They can be converted to monomial form for display or communication.

  • Input Domain None. The function will use the given x-values without modification.
  • Resistance to Outliers Good. Hermite polynomials are less sensitive to outliers due to their orthogonal nature.
  • Numerical Stability High. Hermite polynomials are orthogonal and well-conditioned, making them more stable for high-degree fits, and their lack of a finite domain makes them suited for large input ranges.
  • Calculus Support Supports derivativation, but not integration. Can be converted to monomial form. Root finding uses numerical approximation. See [features#calculus].
  • Orthogonal Yes. See [orthogonality].
Fourier Basis
T[ 0..1000 -> 0..2π ], y(x) = 8.16e7·cos(2xₛ) - 3.05e8·sin(2xₛ) + 3.05e8·cos(xₛ) - 5.46e8·sin(xₛ) + 4.95e8

Fourier polynomials are trigonometric polynomials composed of sine and cosine functions. They have a repeating, curvy appearance and are very well suited to data that tends to be periodic, like signals data or seasonal trends.

They are orthogonal, providing good numerical stability and resistance to outliers. Their periodic nature means they can model cyclical patterns that would require a far higher degree polynomial with other bases.

They have a specific input domain of [0, 2π], so input data must be scaled and shifted accordingly. However, they cannot be converted to monomial form.

  • Input Domain [0, 2π] — input x-values are scaled and shifted to fit within this range.
  • Resistance to Outliers Fair. They can get a decent fit in very noisy data that other bases would struggle with, but may overreact to extreme outliers, and can have artifacts called "ringing" at the edges if used on the wrong types of data.
  • Numerical Stability High. Fourier polynomials are orthogonal and well-conditioned, making them more stable for high-degree fits.
  • Calculus Support Supports full derivativation and integration. Cannot be converted to monomial form. Root finding uses numerical approximation. See [features#calculus].
  • Orthogonal Yes. See [orthogonality].
Logarithmic Basis
y(x) = 1.79e9·ln(x)⁴ + 1.99e9·ln(x)³ - 6.05e8·ln(x)² - 4.70e8·ln(x) + 1.20e8

Logarithmic polynomials are polynomials where the variable is transformed using the natural logarithm function (ln). They are useful for modeling data that exhibits exponential growth or decay, as well as data that spans several orders of magnitude.

They are useful for modelling data with exponential growth or decay, such as population growth, timing data, or financial data. They can capture rapid changes in the data more effectively than standard polynomials.

  • Input Domain (0, ∞) — input x-values must be positive, so the domain does not need to be scaled.
  • Resistance to Outliers Fair. Logarithmic polynomials can be sensitive to outliers, especially if the outliers are extreme or located at the edges of the input range.
  • Numerical Stability Fair. Logarithmic polynomials can be less stable than other bases, especially for high-degree fits or when the input data spans several orders of magnitude.
  • Calculus Support Calculus and Monomial conversion not supported. Root finding uses numerical approximation. See [features#calculus].
  • Orthogonal No. Logarithmic polynomials are not orthogonal.

Orthogonality
Wikipedia | Mathworld | Khan Academy

A set of basis functions is considered orthogonal if each individual basis function does not correlate with the others (adjusting one has no effect on the others' contribution to the total function).

Orthogonal bases tend to have better numerical stability, especially for high-degree polynomial fits.

For polyfit, this means the basis implements the [basis::OrthogonalBasis] trait.

Curves and functions using orthogonal bases can use several advanced functions (same set available on [CurveFit]):
